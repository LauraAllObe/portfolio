Combined Models ML
Jupyter Notebook, Python, Pandas, Scikit-learn, Matplotlib, SMOTE, Logistic Regression, Decision Trees, Random Forests, Gradient Boosting, Stacking Classifier

Application Overview

This project involved implementing multiple machine learning models and combining them using a stacking ensemble technique. A variety of classifiers, including Logistic Regression, Decision Trees, Random Forests, and Gradient Boosting, were stacked to improve accuracy. The notebook also includes bias-variance decomposition to evaluate model performance, with visuals showcasing trade-offs between bias and variance, as well as model accuracy. The best model combination, Logistic Regression with Random Forest Classifier, achieved a mean accuracy of 98.66% with a Bias^2 + Variance value of 0.5. The project culminates in predicting bankruptcy cases and submitting results in CSV format.

Hacker News Web Application
Python, HTML5, CSS, JSON, SQLite, Auth0, Nginx, Gunicorn, Linode, Lynis

Application Overview

The HackerNews Flask application provides a robust and secure platform for delivering the latest news from the Hacker News website, updated hourly. Utilizing nginx and gunicorn for its configuration and SQLite for data management, it offers a streamlined user experience with features like pagination, and user interaction through likes and dislikes for logged-in users. Authentication is managed via Auth0, enabling users to sign in with Google. The application includes several functional routes, such as a profile view for account information and an admin interface for managing user roles. Continuous updates and user interactions are facilitated on a cleanly designed interface, ensuring that users remain engaged and informed.

Likes and Dislikes

Registered users have the ability to like or dislike individual news items. When a user interacts with a news item, their selection is highlighted in red, while unselected options appear in brown. Each interaction updates the total count of likes or dislikes associated with the news item in real time. If a user reselects a highlighted (red) option, it will deselect their like or dislike. Alternatively, if a user selects the opposite option (i.e., liking an item they had previously disliked or vice versa), the system automatically transfers their preference. This user-specific data persists in the database until the user's account is deleted, at which point all associated likes and dislikes are also removed.

Sorting and Pagination

News items are primarily sorted by the number of likes, from highest to lowest. In cases where items have the same number of likes, they are further sorted by their publication time, from newest to oldest. To facilitate easier viewing, these items are paginated, with each page displaying up to 10 news items. Users can navigate through the pages either by using the page navigation controls located at the bottom of the page or by directly specifying the page number in the URL.

Automatic Updates

The system automatically refreshes news items hourly, retrieving the latest 50 unique entries from the Hacker News Firebase API. To ensure users receive up-to-date information, the top 30 items are displayed on the newsfeed page in JSON format, providing a streamlined view of the latest developments.

Admin Permissions

Users with administrative privileges can access the admin page, either by entering the URL directly or via the navigation bar. On this page, administrators have the capability to manage user roles. They can remove non-administrative users as well as add or remove administrative rights for existing users. It is important to note that altering administrative rights does not delete the user's account; it merely changes their access permissions within the system.

User Login

This feature allows users to sign in using their Google account credentials. Upon their initial login, user data is retrieved and automatically added to the Hacker News database. New users will appear in the "All-time users" list accessible from the sidebar.

User Account

Once logged in, users have the ability to access their personal account page.

Sidebar

A list of the all-time users, as well as the UTC time and developer notes can be found in the hacker news sidebar.

FAT32 Filesystem

Program Overview

This program facilitates the manipulation of FAT32 images through a shell-like interface, supporting commands such as ls, cd, mkdir, creat, open, close, read, append, lseek, lsof, rm, and rm -r.

Mount Image

When the program is started, initial values of the image are printed after mounting is complete.

Changing Directory and Making a File

The ls command is used to list the contents of a directory. To navigate between directories, the cd command is utilized, with support for '.' (current directory) and '..' (parent directory) as arguments. Files can be created within the current directory using the creat command. Each command accepts a maximum of one argument except for 'ls' which accepts none.

Making a Directory

The mkdir command is used to make a directory and accepts only one argument. Note that file and directory names must follow 8.3 format guidelines.

Remove and Recursive Remove

The rm command deletes files and directories. To remove directories recursively, the -r flag must be specified as the first of up to two allowed arguments.

File Operations 1/2

The following file operations are available: open requires two arguments, close requires one argument, lsof takes no arguments, lseek requires two arguments, read requires two arguments, and append also requires two arguments. For operations involving an offset, such as lseek, the offset cannot exceed the file's length.

Elevator Kernel Module
C, Kernel Programming, Systems Programming

Program Overview

This program utilizes an advanced elevator scheduling algorithm that interfaces seamlessly with a custom kernel module. It incorporates system calls, efficient scheduling techniques, kernel operations, mutexes/locks, and multithreading to optimize performance and ensure robust system integration.

Elevator Kernel Module

This program, a custom elevator kernel module, utilizes consumer and producer processes to manage start and stop functions. The proc_read function reports the current operational status. Passengers are serviced in a FIFO (First In, First Out) order, with a capacity limit of 750 lbs or five passengers. Once this limit is reached, the elevator will unload but not accept new passengers until capacity is available. The elevator operates in several states: OFFLINE, IDLE, LOADING, UP, and DOWN, and can transport passengers classified as FRESHMAN, SOPHOMORE, JUNIOR, and SENIOR, weighing 100, 150, 200, and 250 lbs respectively. An IDLE state indicates no current passenger service but readiness for input, while OFFLINE indicates a system shutdown via a custom system call.
