Hacker News Web Application
Python, HTML5, CSS, JSON, SQLite, Auth0, Nginx, Gunicorn, Linode, Lynis

Application Overview

The HackerNews Flask application provides a robust and secure platform for delivering the latest news from the Hacker News website, updated hourly. Utilizing nginx and gunicorn for its configuration and SQLite for data management, it offers a streamlined user experience with features like pagination, and user interaction through likes and dislikes for logged-in users. Authentication is managed via Auth0, enabling users to sign in with Google. The application includes several functional routes, such as a profile view for account information and an admin interface for managing user roles. Continuous updates and user interactions are facilitated on a cleanly designed interface, ensuring that users remain engaged and informed.

Likes and Dislikes

Registered users have the ability to like or dislike individual news items. When a user interacts with a news item, their selection is highlighted in red, while unselected options appear in brown. Each interaction updates the total count of likes or dislikes associated with the news item in real time. If a user reselects a highlighted (red) option, it will deselect their like or dislike. Alternatively, if a user selects the opposite option (i.e., liking an item they had previously disliked or vice versa), the system automatically transfers their preference. This user-specific data persists in the database until the user's account is deleted, at which point all associated likes and dislikes are also removed.

Sorting and Pagination

News items are primarily sorted by the number of likes, from highest to lowest. In cases where items have the same number of likes, they are further sorted by their publication time, from newest to oldest. To facilitate easier viewing, these items are paginated, with each page displaying up to 10 news items. Users can navigate through the pages either by using the page navigation controls located at the bottom of the page or by directly specifying the page number in the URL.

Automatic Updates

The system automatically refreshes news items hourly, retrieving the latest 50 unique entries from the Hacker News Firebase API. To ensure users receive up-to-date information, the top 30 items are displayed on the newsfeed page in JSON format, providing a streamlined view of the latest developments.

Admin Permissions

Users with administrative privileges can access the admin page, either by entering the URL directly or via the navigation bar. On this page, administrators have the capability to manage user roles. They can remove non-administrative users as well as add or remove administrative rights for existing users. It is important to note that altering administrative rights does not delete the user's account; it merely changes their access permissions within the system.

User Login

This feature allows users to sign in using their Google account credentials. Upon their initial login, user data is retrieved and automatically added to the Hacker News database. New users will appear in the "All-time users" list accessible from the sidebar.

User Account

Once logged in, users have the ability to access their personal account page.

Sidebar

A list of the all-time users, as well as the UTC time and developer notes can be found in the hacker news sidebar.

My Notes Mobile Application
Java, XML, Android SDK, Room DB, MVVM, LiveData, Androidn Nav Component, Fragments, Navigation Graph, Recycler View, Data Passing with Bundles

Application Overview

This Android application provides comprehensive note management and customization features. Users can adjust text color, background color, font size, and text alignment. It supports saving, deleting, and undoing changes to notes, which can be displayed individually or in a card/recycle view format.

Note Details

The note detail view includes a back button, title and content input fields, and customization options such as text color, background color, text alignment, and adjustable font size in increments of 10s and 1s. It also features buttons for note management, including discard, undo, and save actions. Note that upon exiting, the note content will be automatically saved.

Color Selection

A third-party color selection wheel is utilized to choose both text and background colors. Users can select a color and confirm with "OK" or cancel the selection.

Font Size

Font size adjustments are made using up and down arrows next to the 10s or 1s buttons, allowing selection within a range of 8 to 98. In the notes overview, font sizes are displayed as follows: sizes above 80 appear as 28, above 60 as 23, above 40 as 18, above 20 as 13, and all others as 8.

Text Alignment

Users have the flexibility to align text left, right, or center through the text alignment button conveniently located at the bottom of the screen.

Scrolling and Note Overview

The notes overview utilizes a recycle view, allowing users to add and effortlessly scroll through numerous notes. Each note is presented in a card view, maintaining the same customizations and content as seen in the detail view, with the exception of slightly adjusted font sizes.

ProFessUp Web Application

Application Overview

ProFessUp is a platform modeled after Rate My Professor, designed to facilitate CRUD operations for professors, reviews, courses, and user profiles. The application enables users to efficiently search for professors and filter reviews by course. It presents professor profiles and reviews in a card-based layout, utilizing sliders, drop-down menus, buttons, and checkboxes to enhance usability and comprehension. The interface employs a user-friendly color scheme to optimize the user experience. Due to the deletion of the MongoDB, note functionality is demonstrated exclusively through video in this portfolio project.
